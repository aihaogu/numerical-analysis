# Chapter 02 实验报告

## 摘要
本实验针对教材第二章给出的两个线性方程组，分别采用普通高斯消去法与列主元高斯消去法求解，比较不同主元策略对数值稳定性的影响。实验结果显示，在两个条件良好的测试矩阵上，两种方法均获得一致解向量，说明在主元足够大的情况下普通高斯消去法也能保持稳定；同时也进一步强调了列主元策略在面临潜在病态问题时的重要性。

## 前言（目的和意义）
本实验旨在通过动手实现高斯消去类算法，加深对线性方程组数值解法的理解。具体目标包括：

1. 熟悉高斯消去法及其列主元变体的算法流程；
2. 评估主元策略对求解精度和稳定性的影响；
3. 为后续分析更高维或病态系统的数值行为奠定基础。

线性方程组求解是科学与工程计算的核心问题，高效稳定的求解器直接影响后续仿真与优化的可靠性。因此，通过对比不同消去策略的表现，可以为算法选择提供经验依据。

## 数学原理
高斯消去法通过对增广矩阵执行行初等变换，将系数矩阵化为上三角形式，然后利用回代得到解向量。其关键步骤包括：

1. 选取当前列的主元；
2. 对主元所在行以下的行执行消元，消除该列的其余元素；
3. 完成消元后从最后一行开始回代求解。

普通高斯消去法默认使用对角线上元素作为主元，当主元接近零时可能放大舍入误差；列主元高斯消去法在每一列选择绝对值最大的元素作为主元，通过行交换提高数值稳定性。两者理论上在精确算术下等价，但在浮点运算中列主元策略能显著降低误差积累。

## 程序设计流程
实验采用 Python 编写，`problem1.py` 与 `problem2.py` 分别对应两个线性系统。每个脚本均包含以下流程：

1. **数据准备**：以 NumPy 数组形式定义系数矩阵 $A$ 和常数向量 $b$；
2. **算法实现**：分别实现不带主元和带列主元的高斯消去函数，函数内部对零或过小主元进行检查以避免除零；
3. **求解与比较**：调用两种算法获得解向量，计算两种解之间的分量差异；
4. **结果输出**：格式化打印解向量、差异以及算法执行过程中的关键信息。

## 实验结果、结论与讨论
两个测试系统的计算结果如下（数值保留十位小数）：

- **问题（1）**
  - 高斯消去法：$x = (0.7037564870,\ 1.9002387860,\ 0.1471716371)$
  - 列主元高斯消去法：$x = (0.7037564870,\ 1.9002387860,\ 0.1471716371)$
  - 解向量分量差异：$0$
- **问题（2）**
  - 高斯消去法：$x = (2.0000000000,\ 3.0000000000,\ 3.0000000000)$
  - 列主元高斯消去法：$x = (2.0000000000,\ 3.0000000000,\ 3.0000000000)$
  - 解向量分量差异：$0$

两个系数矩阵的主元均远离零，且不存在明显病态特征，因此普通高斯消去法即可获得稳定结果。列主元策略在这些案例中未展现出优势，但其行交换步骤仍是处理病态或数值不稳定系统时的必要手段。综合实验可得结论：在条件数适中的系统中，普通高斯消去法足以满足精度需求；然而为了提高鲁棒性，实际工程中仍建议优先采用带主元策略。
